2. 1주차 주요 개념 키워드 적어보기
-버전관리 : 프로젝트 상태가 변경되는 정보를 알고있다.
-git 초기화(init) : 컴퓨터에 만든 폴더를 git 으로 관리하겠다고 선언?설정해주는 작업
-commit : 작업 내용을 저장(누가/언제/변경사항 등을 작성하는게 좋은 커밋 메세지)
-add(staging) : 소스트리에서 commit 내용을 로컬리포에 반영할지 선택해주는다.
(init>파일 내용 작성>add>commit 메세지>commit)

-repo : repository(저장소)라고 불리며 로컬 리포, 원격 리포 라고 부른다.
원격 리포 는 네이버 클라우드, 구글 드라이브 같은 것이며 대표적으로 깃허브가 있다.
로컬 리포 는 내 pc의 저장소 이다.
원격리포,로컬 리포가 분릴되어있는 이유는? 협업 때문이다!
원격 리포는 여러명이 접근할 수 있기에 개인의 작업을 원활히 할 수 있기 위함이라고 생각한다.

-traching: 로컬 리포 와 원격 리포를 연결 시켜준다.
(소스트리 사용법 : 깃허브 리포지토리 생성>url 복사>소스트리 설정>url 붙여넣기)
-push : 로컬 리포에 작업 반영사항을 원격 리포에 밀어넣는다.
-pull : 원격 리포에 작업 반영사항을 로컬 리포로 내려받는다.
-clone : 원격 리포를 내 컴퓨터에 가져와 복제한다.(새로투입되는 프로젝트 경우, 내 pc에는 그 플젝 파일이 없다!)
